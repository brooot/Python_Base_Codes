
http
http协议　　　　　是超文本传输协议　　应用层协议　　传输层使用TCP
用途：互联网中网站的数据传输　
特点：
１．支持客户端，服务服模式
２．灵活简单快速
http协议几乎可以传输任意格式的数据
３．http1.o无连接   最新的是http1.1支持持久连接
４．无状态：　　不记录用户的访问信息

数据传输规：
http 请求
    格式:
        请求行
        请求报头
        [空行]
        请求正文
    请求方法：
    GET   获取URI所标识的所有资源
    POST　　提交一定附加数据用以获取相应的返回
    HEAD　　获取URI所标识的所对应消息报头
    PUT　　　获取服务器的一个存储资源
    DELETE　删除URI所标识的服务器端资源
    TRACE　　用于测试和诊断
    CONNECT　保留将来用
    OPTIONS　请求服务器新能，用来查询服务器资源相关的信息
    请求行格式：　GET/index.html HTTP/1.1

按Ctrl+Shift+i  查看元素

http 响应
    响应行
    响应报头
    [空行]
    响应正文
响应码
1xx 提示信息，标识请求已经接受，正在处理
2xx 成功访问已经成功，正确解析受理
3xx 重定向　　完成任务要进行进一步的请求操作
4xx 客户端错误
5xx 服务器端错误

200 ok 表示访问成功，
400 Bad Request 客户端请求语法有误，
服务器不理解，
401 请求未授权，
403　服务器收到请求，但是拒绝提供服务
404　请求的资源不存在或URL错误
500 服务端发生不可预期的错误
503 服务器暂时不能处理请求，一段时间后可能可以

响应行格式：HTTP/1.1  200 ok
响应报头格式：同请求报头
响应内容：将客户端请求得到的内容发送给客户端

sig：要发送什么信号

处理信号
import signal
主要使用signal函数
signal.signal(sig,action)
功能：在进程中异步的处理信号

当进程设定了信号处理后，不会阻塞而是交给内核帮助记录，进程继续执行下一步，
当进程结束前任意时刻收到信号就处理

参数
sig:要处理的信号
action: 处理方法，
有三种情况
SIG_IGN  (忽略该信号)
SIG_DFL  (采用默认处理)
function　　(采用指定的处理方法)
