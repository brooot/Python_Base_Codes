４．MySQL的特点
１.关系型数据库
　　　数据是以行和列形式去存储的
　　　这一系列的行和列称为表，表中每一行叫一条记录，
　　　表中每一列叫一个字段，表和表之间的逻辑关联叫关系
　　　关系型数据库的核心内容是关系既二维表
　　　
２．跨平台
　　　可以在Unix,Linux Windows
３．支持多种编程语言
　　　python ,java,

 数据类型：
 １．数值类型
 　　　1.int 大整型（４个字节）
 　　　存储范围０～２**32
    tinyint 微小整型
    　　　有符号（－１２８～１２７）
    　　　无符号unsigned 0~255
    　smallint 小整型（２个字节）
　　　　　bigint  极大整型　８字节　２**64　
    　2.浮点型
    　　　float (4 个字节)
    　　　用法：字段名　float(m,n) #m表示总位数，n小数位位数
　　　　　　　注意：浮点型插入整数时，mysql会自动补全小数点位数，
　　　　　　　如果插入数据时，小数位数多余指定位数，会对指定位数的下一位进行四色五入
　　　　　　　double(8个字节)
　　　　　　　　　字段名　double(m,n)
　　　　　　　decimal
         字段名　decimal(m,d)
         m>d   m+2个字节的存储空间
         m<d   d+2个字节的存储空间
       #数值类型的宽度为显示宽度，只用于select查询显示和占用存储空间大小无关
       用zerofill展示效果，字符类型的宽度超过则无法存储  
 ２．字符类型
　　　　１．char(定长)　　　＃性能高　浪费存储空间
　　　　　　　取值范围1~255个字符
　　　　２．varchar(变长)　　性能低　　节省存储空间
　　　　　　　取值范围1~65532 
 ３．枚举类型：　
 　　　　　定义：字段只能在列表范围内选择
 　　　　１．enum 单选
 　　　　　　语法：字段名　enum(值１...)
     2.set 多选
     　　语法：字段名　set(值1,值２...)

６．表字段的操作：
语法：alter table 表名　执行动作；
　１．添加一个字段
　　　　alter table 表名　add　字段名　数据类型；
　　　　alter table t20 add address varchar(30);
2.添加到指定位置
　　　　alter table t20 add address varchar(30) after 字段名；
3.删除
　　　　alter table t20 drop 字段名　
４．修改字段的数据类型：
alter table 表名 modify 字段名　新的数据类型;
alter table t20 modify score　tinyint;
